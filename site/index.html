Branch map

Each branch has a clear responsibility and reduces overlap.

  • main

    Stable release branch protected for publishing.

  • dev

    Integration branch for day-to-day work and release prep.

  • feature/layout

    Page structure, sidebar, cards, and the visual graph.

  • feature/content

    Report text, answers, and timeline content.

  • hotfix/deploy

    Final deploy path fixes and GitHub Pages details.

Deployment

Ready workflow for automatic deployment after pushing to main.

  1. Check out the code with actions/checkout
  2. Configure GitHub Pages with actions/configure-pages
  3. Upload the `site/` folder as the artifact
  4. Deploy with actions/deploy-pages
