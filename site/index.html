نقشه‌ی branchها

هر شاخه یک مسئولیت واضح دارد و تداخل را کم می‌کند.

  • main

    نسخه‌ی پایدار و protected برای انتشار.

  • dev

    شاخه‌ی ادغام کارهای روزمره و آماده‌سازی release.

  • feature/layout

    ساختار صفحه، sidebar، cards و visual graph.

  • feature/content

    محتوای گزارش، answerها و timelineهای متنی.

  • hotfix/deploy

    رفع نهایی مسیر deploy و جزئیات GitHub Pages.

استقرار

Workflow آماده برای deploy خودکار بعد از push به main.

  1. Checkout کد با actions/checkout
  2. تنظیم GitHub Pages با actions/configure-pages
  3. Upload پوشه‌ی site به عنوان artifact
  4. Deploy با actions/deploy-pages
